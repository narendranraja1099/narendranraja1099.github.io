About

Engagement Areas &
Professional Background.

5+ years of hands-on offensive security experience across web, mobile, API, cloud, automotive, and embedded domains. Not a generalist — a practitioner who operates with depth across each attack surface.

Primary focus spans enterprise-grade web application, mobile (Android & iOS), API, and cloud security assessments — delivered across multiple client environments in the IT sector. Methodology is manual-first with targeted automation. The goal is to find what scanners miss, not to validate what they already flag.

Secondary specialization in automotive cybersecurity — working at the protocol level across CAN, LIN, and Automotive Ethernet; testing ECUs under black-box and grey-box conditions; and applying firmware reverse engineering where conventional tooling is insufficient. This domain experience is a differentiator, not a side note.

Findings are documented to drive real remediation — clear reproduction steps, risk-prioritized impact, and developer-ready mitigations. Every engagement closes with actionable output, not padded reports. Standards aligned: OWASP, PTES, and ISO/SAE 21434 where applicable.

Services

What I Test.

Each engagement is scoped, executed, and reported with precision. No templates. No automated-only outputs.

01
Web Application Security
OWASP-based manual testing with deep business logic analysis. Finding what automated scanners consistently miss — across single domains and multi-subdomain scopes.
  • OWASP Top 10 full coverage
  • Business logic and workflow exploitation
  • Authentication bypass, session fixation, CSRF
  • SSRF, XXE, SQLi, Blind SQLi, XSS, SSTI
  • Insecure Direct Object Reference (IDOR)
  • File upload bypass and path traversal
  • Subdomain enumeration and attack surface mapping
  • Clickjacking, open redirect, security misconfiguration
02
Mobile Application Security
Static and dynamic analysis on Android and iOS applications. Runtime instrumentation, data exposure, insecure backend coupling, and platform-specific attack surface coverage.
  • Static analysis — APK/IPA decompilation and code review
  • Dynamic analysis — Frida instrumentation and runtime hooking
  • SSL pinning and root / jailbreak detection bypass
  • Hardcoded secrets, tokens, and insecure local storage
  • Improper session handling and authentication flaws
  • Deep link and intent hijacking (Android)
  • WebView attack surface and JavaScript bridge abuse
  • OWASP Mobile Top 10 / MASVS aligned
03
API Security Testing
REST and GraphQL endpoints tested for broken access control, mass assignment, data exposure, and abuse-path exploitation. Covers both authenticated and unauthenticated attack surfaces.
  • Broken Object Level Authorization (BOLA / IDOR)
  • Broken Function Level Authorization
  • Broken authentication and JWT flaws
  • Mass assignment and parameter pollution
  • GraphQL introspection, batching, and injection
  • Excessive data exposure and response filtering bypass
  • Rate limiting bypass and resource abuse
  • OWASP API Security Top 10 coverage
04
Cloud Security Assessment
Misconfiguration audits and privilege escalation path analysis across AWS, GCP, and Azure environments.
  • Cloud misconfiguration review (AWS/GCP/Azure)
  • IAM privilege escalation path enumeration
  • Exposed storage and service discovery
  • Network segmentation and trust boundary review
  • Risk-based remediation prioritization
05
Automotive Security Testing
ECU-level penetration testing across multiple automotive communication protocols and architectures. From diagnostic interfaces to telematics attack surfaces.
  • ECU penetration testing (TCU, PEPS, clusters, cameras, LiDAR, RADAR)
  • CAN, LIN, and Automotive Ethernet — protocol and ECU level assessment
  • UDS diagnostic interface fuzzing, secure access, and secure boot analysis
  • Firmware reverse engineering — ARM, RH850, and x86 architectures
  • Spoofing, replay, and DoS at the protocol layer
  • ISO/SAE 21434 aligned assessments and TARA
06
IoT & Embedded Device Security
Device-level attack surface analysis covering firmware extraction, debug interface exposure, and insecure communication channels.
  • Firmware extraction and static binary analysis
  • JTAG/UART debug interface enumeration
  • Hardcoded credential and secret discovery
  • Insecure update mechanism analysis
  • Cloud backend coupling and trust boundary review

ECU Security Assessment
Grey-box protocol-level and diagnostic-layer penetration test of a production-bound automotive ECU. Testing conducted on an embedded test bench under pre-OEM integration conditions with access to ECU documentation and signal databases.

Automotive ECU (OEM confidential)

~25 hrs
